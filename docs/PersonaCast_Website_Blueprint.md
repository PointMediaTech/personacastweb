### 1.2.2 全站 SEO 技術規範

- **URL 結構**：全站使用靜態化、語意化路徑 (`/solutions/crisis-pr` 而非 `/page?id=3`)
- **Schema Markup**：首頁 `Organization`、產品頁 `SoftwareApplication`、部落格 `Article`、FAQ `FAQPage`
- **Open Graph / Twitter Card**：每頁自訂 OG 圖與描述
- **Core Web Vitals 目標**：LCP < 2.5s、CLS < 0.1、INP < 200ms
- **語系**：zh-TW 為主站 (hreflang)，預留 en 國際版結構
- **XML Sitemap**：自動生成、提交至 Google Search Console
- **Canonical Tags**：所有頁面設定自引用 canonical
- **Internal Linking**：各解決方案頁 ↔ 對應產品功能頁 交叉連結；部落格文章底部嵌入相關解決方案 CTA

## 2.19 全站轉換漏斗設計總結

```
                     ┌─────────────┐
         TOFU        │   Blog /    │  ← SEO 長尾流量
      (Awareness)    │  Resources  │  ← 社群分享
                     └──────┬──────┘
                            │
                     ┌──────▼──────┐
         MOFU        │  Solutions  │  ← 場景共鳴
     (Consideration) │  Product    │  ← 功能說服
                     │  Case Study │  ← 社會證明
                     └──────┬──────┘
                            │
                     ┌──────▼──────┐
         BOFU        │  Pricing    │  ← 方案比較
      (Decision)     │  Contact    │  ← Demo 預約
                     │  Whitepaper │  ← Lead Magnet (Email Gate)
                     └─────────────┘
```

**每一頁的轉化路徑設計原則**：
1. 頁面頂部（Hero）：價值主張 + Primary CTA（預約 Demo）
2. 頁面中段：教育/說服內容 + 場景連結 + Social Proof
3. 頁面底部：強化 CTA + 替代路徑（白皮書下載 / 部落格 / 案例）
4. 全站固定 CTA：右下角浮動「預約 Demo」按鈕（scroll 超過首屏後出現）
